## 一、核心定位

**工业AI落地操盘手 — 能把模糊的智能化愿景拆解成可量化回报的落地方案**

稀缺性来源：市场上"懂工业场景 + 会用AI创造可量化经营价值"的人极少。大多数人要么只懂技术不懂业务，要么只懂业务不懂AI。

### 市场人才分布

- **纯AI技术人才**（会调模型、写算法）— 供给充足，竞争激烈
- **纯工业/制造人才**（懂工艺、懂产线）— 传统行业大量存在
- **能把AI落地到工业场景并量化经营价值的人** — 极度稀缺（你在这里）

## 三、问题诊断

### 核心结论：不是能力问题，是土壤问题

当前困境的三重结构性原因：

1. **AI是支撑职能而非收入引擎** — 业务部门的KPI不跟你的系统挂钩，没动力配合改流程
2. **数据基建无主** — 高速成长期的私企，系统是补丁摞补丁，没有统一的数据治理归属
3. **大组织惯性** — 业务方要的是AI打杂，不是AI重塑工作流；人性抵抗"被重塑"

这不是理想一家的问题，是"AI作为赋能部门"在所有大公司的普遍困境。

### 瓶颈的两个具体表现

1. **需求侧的认知错位** — 业务方把AI当"高级助手"（查数、打杂），不愿为数据闭环投入改变工作流。建立一套准确完整的数据闭环、服务于AI模型迭代和经营价值积累，跟人性多少有冲突。
2. **供给侧的基建缺失** — 公司内IT系统林立，数据基础混乱，缺少高可信的数据基建。这件事不属于任何业务部门的职能范围，推动协作困难。为了做一个智能化应用，要从设备维度、指标、检测数据等最基础环节开始搭建，周期长、阻力大，且私企有快速交付压力。

### "土壤问题"深度分析：普遍性 vs 个例性

#### 矛盾1：AI需要"流程重塑"，但人性抵抗"被重塑"（普遍问题）

| 环境特征 | 阻力大 | 阻力小 |
|---------|--------|--------|
| AI与核心业务的关系 | AI是"辅助工具"，业务可用可不用 | AI就是业务本身，不用就没法运转 |
| 谁在推动 | 中层技术团队自下而上推 | CEO/创始人自上而下强推 |
| 利益结构 | AI提效 = 有人的活被替代 = 被抵制 | AI提效 = 大家都赚更多钱 = 被欢迎 |
| 组织规模 | 大组织有大量"信息中间商"会被压缩 | 小组织本来就没有冗余层 |

**在理想汽车，AI是支撑职能而非收入引擎。** 部门再强，也是"帮别人做事"的角色。业务部门的KPI不跟你的系统挂钩，自然没动力配合改流程。

#### 矛盾2：做AI需要好的数据地基，但数据地基"不归任何人管"

| 公司类型 | 数据基建状态 | 原因 |
|---------|------------|------|
| 高速成长期的私企（如理想） | 混乱 | 业务跑太快，系统是补丁摞起来的 |
| 成立>15年的大国企 | 也混乱，但有预算治理 | 历史包袱重，但有专项资金做数据中台 |
| 数据原生公司（如SaaS/AI公司） | 相对干净 | 数据本身就是产品，从第一天就认真对待 |
| 新成立的小公司 | 最干净 | 没有历史包袱，从零搭可以一步到位 |

### 关键短板

- **独立产品定义力**：一直在执行老板的产品思路，缺少从零定义产品并为结果负责的经验
- **对外价值表达力**：有EVI量化能力，但缺少面向客户/投资人讲价值故事的经验

## 四、目标方向分析

### 稳定性优先级排序

行业确定性 > 公司确定性 > 节奏确定性（兼顾家庭）> 成长确定性

### 对企业形态的判断

小公司也是一种选择。价值观正确、长期主义的小公司，可能才是未来最佳的企业形态。有AI加持就不太需要规模很大，现在大公司的很多岗位和职能会逐步缩减。

### 方向A：早期团队/小公司（首选）

目标角色：AI Product Lead / 技术合伙人（场景侧）

前提条件：工作内容能延续之前的经验优势，且相对稳定、长期主义。

| 要求 | 匹配度 | 差距 |
|-----|--------|-----|
| 独立定义产品方向 | 60% | 小公司的"整盘"往往就是一个垂直场景，子系统经验可能够用 |
| 对外讲价值故事 | 50% | 需要刻意练习 |
| 带5-15人小团队交付 | 85% | "连长"经验的甜蜜点 |
| 懂技术架构不需全写 | 80% | 架构拆解+AI工具辅助开发的组合匹配 |
| 垂直行业深度认知 | 90% | 汽车制造/供应链质量是主场 |

优势：人际简单、快反馈、高天花板
风险：公司可能活不久

### 方向B：大国企数智化研究院（备选）

目标角色：AI解决方案架构师 / 智能化咨询专家

| 要求 | 匹配度 | 差距 |
|-----|--------|-----|
| 输出方法论和白皮书 | 70% | 有实战积累，缺系统化输出习惯 |
| 跨行业智能化规划 | 40% | 目前只深耕汽车 |
| 长周期研究耐心 | 70% | 性格偏长期主义 |
| 学历背景 | 95% | Georgia Tech硕士，海归背景加分 |
| 学术产出 | 40% | 有两个专利初稿，论文/标准偏少 |

优势：稳定、长期、不焦虑
风险：可能陷入"做PPT"的虚耗

### 两条路的核心差异对比

| | 早期团队 | 国企研究院 |
|--|--------|-----------|
| 你需要补的 | 产品定义力 + 对外表达力 | 体系化输出 + 跨行业视野 |
| 匹配你性格的 | 小团队、人际简单、快反馈 | 稳定、长期、不焦虑 |
| 风险 | 公司可能活不久 | 可能陷入"做PPT"的虚耗 |
| 天花板 | 如果公司成了，你就是核心高管 | 高级专家/总工程师 |

## 五、选公司"五问检验法"

面试/交流时必问：

### 1. AI是成本中心还是利润中心？

- **成本中心**（帮内部提效）→ 永远在"求着别人用" → 大概率重蹈覆辙
- **利润中心**（对外卖AI能力/产品）→ 工作直接产生收入 → 话语权完全不同

> 关键问话："你们AI团队的成果，最终体现在公司的哪条收入线上？"

### 2. 数据决策权归谁？

- 如果"数据归各业务部门各管各的" → 又要花80%精力协调数据
- 如果"有统一的数据平台/团队，且有CTO/CDO级别的人负责" → 能专注在AI应用层

> 关键问话："如果我需要一个新数据源接入，流程大概多久？谁来决策？"

### 3. 推动变革的力量来自哪里？

- 如果是中层技术团队在推 → 跟在理想一样，上面不给力
- 如果是创始人/CEO直接驱动 → 阻力会被权力势能压平

> 关键问话："你们CEO/创始人平时会参与AI相关的产品决策吗？多频繁？"

### 4. 组织对"长周期投入"的容忍度

- 如果要求"每个季度出数字" → 又被迫做短平快的"查数打杂"类需求
- 如果能接受"前6个月搭基建，后面持续产出" → 才能做对的事

> 关键问话："你们现在最长的一个AI项目从立项到产出价值花了多久？中间有被砍掉的压力吗？"

### 5. 团队里谁懂业务？

- 如果全是纯技术人 → 一个人既要翻译业务又要落地技术，累死
- 如果已有行业专家 + 你来做AI落地桥梁 → 这是最佳位置

> 关键问话："你们团队里有没有从产线/业务一线转过来的人？"

## 六、半年行动计划

### 主线：补产品定义力

用"如果我是创始人"的视角重新梳理供应链质量预警体系：目标客户、价值主张、MVP、商业模式、定价逻辑。输出一份10页以内的产品方案。

这不一定真的去创业，而是训练"从零定义产品"的思维肌肉。输出物在后续做简历/面试时可直接使用。

### 支线1：积累体系化输出

- EVI方法论、本体论实践写成2-3篇结构化文章（公众号/知乎/内部分享都行）
- 既是给国企路线攒素材，也是给小公司路线建立个人品牌
- 两份专利继续推进到授权

### 支线2：接触市场

- 不急着投简历，先以"行业交流"的姿态接触目标圈子
- 关注海淀/昌平片区做工业AI、智能制造、供应链数字化的团队
- 渠道：脉脉行业群、前同事/校友内推、垂直领域会议

### 时间节奏

- 前3个月：补能力 + 建输出 + 广撒网了解市场
- 后3个月：聚焦1-2个机会深度推进
