## 目标岗位分析

### 岗位A：AI产品经理（智能制造方向）

#### 核心职责

1. **场景挖掘** — 深入压铸、组装、质检、物流等产线，发现AI可优化的痛点
2. **产品定义** — 将AI算法应用产品化（视觉质检、工艺参数优化、机器人协同）
3. **效果闭环** — 建立AI模型效果评估体系，推动模型迭代
4. **跨界协作** — 协调算法工程师、自动化设备工程师、产线运营团队

### 岗位B：数字化产品经理

#### 核心职责

1. **业务数字化** — MES/ERP/WMS系统规划与升级
2. **数据驱动** — 构建工厂数据中台，发现生产瓶颈
3. **IT-OT融合** — 推动信息技术与设备控制的深度集成

## 真实工作体验（脉脉/知乎综合）

### ⚠️ 工作强度警告

| 维度 | 评价 |
|------|------|
| 加班文化 | ⚠️ **严重** — 内部存在"日均工时考核"，部分部门要求≥11.5小时 |
| 工时排名 | ⚠️ 工时排名靠后会被约谈、写说明、影响绩效 |
| 出差需求 | 中低（工厂在昌平/亦庄，产品岗主要驻厂而非出差外地） |
| 弹性程度 | 低 — 有工时考核就意味着必须在工位待够时间 |

### 正面评价

| 维度 | 评价 |
|------|------|
| 技术水平 | 国内领先 — 国家级智能制造标杆，"黑灯工厂" |
| 数据基建 | ✓ 好 — 自研"澎湃智能制造平台"，从Day1就重视数据 |
| 成长空间 | 好 — 接触前沿制造技术 |
| 薪资福利 | 有竞争力 — 五险一金全额、免费三餐、班车、宿舍 |
| 技术氛围 | 工程师文化强 |

### 负面评价（高频出现）

- ⚠️ **工时考核** — 这是小米2024-2025年最大的争议点
- ⚠️ 部分部门要求日均14-15小时在岗
- 加班费年底结算/延迟发放
- 虽然是"互联网公司"但制造部门节奏接近传统制造业

### 对宝妈的关键评估

| 维度 | 评价 | 对比 |
|------|------|------|
| 出差压力 | 低（工厂本地，不需全国跑） | ✓ 好于第四范式和三一 |
| 加班压力 | ⚠️ **很高**（工时考核制度） | 与三一差不多 |
| 通勤压力 | 中（昌平，10-15公里） | 不如用友 |
| 工作生活平衡 | ⚠️ **差** — 日均11.5小时在岗几乎没有家庭时间 | 仅好于三一 |

**核心矛盾：** 小米不出差是好的，但日均11.5小时的工时考核 = 每天早8晚8最低限。对于有宝宝需要照顾的你，这几乎不可行。

## 综合评估

### 适合你的理由

1. **数据基建是目前看到最好的** — 自研平台从Day1就做对了，不存在"数据无主"问题
2. **场景高度匹配** — AI质检、预测性维护、产线数据闭环 = 你的老本行
3. **高层真的在推** — 雷军直管，前特斯拉厂长带队
4. **不需要出差** — 工厂在本地，不用全国跑
5. **汽车制造经验直接可用** — 小米汽车工厂的逻辑和理想类似

### ⚠️ 核心风险

1. **工时考核制度** — 日均11.5小时在岗要求 = 每天早8晚8起步。对宝妈几乎是dealbreaker
2. **AI不是利润中心** — 仍然是内部提效，虽然高层推动力强
3. **你的"从0搭建"能力浪费** — 他们数据基建已经很好，不需要你重新搭
4. **大厂内部竞争** — 小米人才密度高，晋升竞争激烈

## 我的判断

> **小米在"场景匹配"和"数据基建"维度上是最优的，但工时考核制度对宝妈是硬伤。**

**如果工时考核可以在面试中确认为"非强制/部门差异大"，那小米值得深度考虑。**

关键验证点：智能制造部（产品岗）是否跟手机互联网部门一样执行11.5小时工时考核？制造部门的节奏可能和互联网部门不同。
