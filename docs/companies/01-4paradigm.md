## 业务架构

### 三大业务板块

```
第四范式
├── Sage AIOS（先知AI平台）— 核心营收引擎（~70%）
│   └── 提供算力层、平台层、模型层、应用层
├── SHIFT（智能解决方案）— 行业场景落地
│   ├── 制造业：产线质检、排程优化、设备预测维护、供应链
│   ├── 能源：管道异常检测、负荷预测、设备故障诊断
│   ├── 金融：风控、反欺诈、精准营销
│   └── 零售：需求预测、库存优化
└── AIGS（AI-Generated Software）— 生成式AI开发工具
```

### 组织模式

- **非传统事业部制**，而是"平台型组织"
- 先知平台团队（底座）+ SHIFT行业解决方案团队（场景）紧密配合
- 以"标杆客户"为导向 — 服务头部企业沉淀行业Know-how，再标准化扩张
- 制造业标杆客户：**宁德时代**（AI质检+排程优化+研发辅助）

## 目标岗位分析

### 岗位：解决方案架构师（工业/制造方向）

#### 核心职责

1. **业务洞察与需求挖掘**
   - 深入理解制造业业务流程、痛点及数字化转型需求
   - 与客户中高层沟通，将业务问题转化为AI解决方案

2. **解决方案设计与规划**
   - 整体架构设计（业务架构、应用架构、数据架构、技术架构）
   - 结合先知平台能力，制定端到端智能化转型方案

3. **技术咨询与方案宣讲**
   - 向客户进行方案宣讲、技术交流、PoC演示
   - 招投标技术方案编写与答辩

4. **项目落地与协同**
   - 协调产品、研发、交付团队确保高质量交付
   - 持续跟踪业务效果，优化模型与方案

5. **行业沉淀**
   - 总结最佳实践，建设行业知识库与标准化方案

| 要求 | 具体内容 | 你的匹配度 |
|------|---------|-----------|
| 学历 | 计算机/自动化/工业工程/数学本科及以上 | ✓ GT EE/CS硕士 |

| 抗压执行力 | 适应高强度，跨部门协作 | ✓ |

#### 你的匹配度总评：75-80%

**核心优势：**
- 6年制造业AI落地实战（远超要求的5年）
- EVI价值量化方法论 = 他们最看重的"可衡量AI价值"
- 供应链质量预警经验 = 直接匹配SHIFT制造方向

**需要补的：**
- 对外客户沟通经验（你的经验是内部推动，不是面对外部客户）
- 系统架构设计的深度技术表达（分布式系统、微服务等技术术语）
- 熟悉第四范式的产品体系（先知平台、式说AIGS等）

### 工作强度

| 工作节奏 | 快，OKR+项目交付双重驱动 |
| 弹性办公 | 官方说有弹性工作时间，实际取决于项目和领导 |

### 文化氛围

| 维度 | 评价 |
|------|------|
| 技术氛围 | 强，创始人技术背景，内部技术讨论活跃 |
| 扁平程度 | 中等（公司宣传扁平，但上市后流程化增加） |
| 成长空间 | 好（能接触前沿AI+复杂企业场景） |
| 内部政治 | 相对简单（技术导向公司） |

### 对宝妈的关键风险点

⚠️ **出差是最大的顾虑**
- 解决方案架构师岗位，面向外部客户 = 必须出差
- 制造业客户多在外地（如宁德时代在福建/四川）
- 项目攻坚期可能连续出差多天

**可能的缓解方案：**
- 面试时明确问"该岗位区域覆盖范围"和"月均出差天数"
- 询问是否有"内部支持型"角色（如方案库建设、内部赋能、远程支持）— 出差频率远低于一线作战型
- 如果负责北京/华北区域客户，出差压力会小很多

## 综合评估

### 适合你的理由

1. **AI是核心业务** — 不再"求人用AI"，而是"客户付费请你帮他们用AI"
2. **你的EVI方法论直接变现** — 他们最看重"可衡量的AI价值"
3. **制造业是重点赛道** — 宁德时代等标杆已跑通，你的经验直接可用
4. **地理位置** — 上地，通勤友好
5. **薪资匹配** — 不降薪换环境的最佳选项之一

### 风险与顾虑

1. ⚠️ **出差问题** — 解决方案岗面向外部客户，出差不可避免。对新晋宝妈是核心矛盾
2. **可能变成"高级售前"** — 如果不注意，角色可能偏向写方案+投标，而非产品定义
3. **上市后增长压力** — 有可能被要求"快速签单"而非"深度服务"
4. **技术深度要求** — 面试时可能考察分布式系统等技术细节，需准备

### 面试时必问的问题

1. "制造业方向的解决方案团队目前多少人？覆盖哪些区域的客户？"
2. "这个岗位月均出差大概几天？是否有远程支持的协作模式？"
3. "你们制造业方向目前最大的挑战是什么？是获客还是交付？"
4. "团队里有从制造业一线转过来的人吗？还是主要是技术背景？"
5. "项目交付的闭环是怎样的？方案设计完之后谁负责落地？"
